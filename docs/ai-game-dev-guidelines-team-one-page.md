# AI를 활용한 서버/클라이언트 개발 가이드라인
## 팀 공유용 1페이지 요약

## 1. 핵심 개념
AI 활용 방식은 두 가지로 나눈다.

- **A:** 빠르게 검증하는 방식
- **B:** 메인 프로젝트에 편입하는 방식

가장 큰 차이는 **문제가 생겼을 때 무엇을 기준으로 수정하느냐**다.

- **A:** 문서를 수정하면서 검증
- **B:** 코드와 구조를 수정하면서 편입

한 줄 요약:
- **A는 검증용**
- **B는 편입용**

---

## 2. 기본 작업 흐름
A와 B 모두 아래 흐름을 기본으로 한다.

1. **요구사항 / 개발 계획을 md 파일로 정리**
2. **md 파일을 기준으로 작은 단위 구현**
3. **AI 리뷰, 인간 리뷰, 테스트 진행**
4. **수정사항 정리 후 다시 1번으로**

---

## 3. A와 B를 나누는 기준

### A를 선택하는 경우
- 빠른 검증이 목적이다.
- 감각, 흐름, 기술 가능성을 먼저 보고 싶다.
- 코드가 나중에 버려져도 괜찮다.
- 문제가 생기면 요구사항, 개발 계획, 테스트 시나리오를 수정해서 다시 확인할 생각이다.

### B를 선택하는 경우
- 메인 프로젝트에 넣어야 한다.
- 다른 개발자와 협업해야 한다.
- 코드 리뷰, 테스트, 유지보수가 필요하다.
- 문제가 생기면 실제 코드와 구조를 분석해서 고쳐야 한다.

---

## 4. A에서 일하는 방식
A에서는 먼저 문서를 보고 고친다.

### A에서 주로 하는 수정
- 요구사항 문구 수정
- 개발 계획 범위 재정리
- 테스트 시나리오 추가
- 테스트용 데이터 / 입력값 조정
- 다시 구현하고 다시 확인

즉 A는 **문서 중심으로 검증을 반복하는 방식**이다.

---

## 5. B에서 일하는 방식
B에서는 기본 작업 흐름에 들어가기 전에 아래 정리가 먼저 필요하다.

1. 기존 프로젝트 기능별 코드베이스 분석
2. 코딩 컨벤션 / 구조 규칙 파악
3. 현재 프로젝트와 대상 모듈 간 차이 정리
4. 마이그레이션 전략 / 요구사항 / 개발 계획 문서화

### B에서 주로 하는 수정
- 실제 코드 패치
- 구조 조정
- 계약 문서 수정
- 테스트 추가 / 수정
- 리뷰 후 재검증

즉 B는 **기존 프로젝트 분석 + 문서화 + 코드 수정**이 함께 가는 방식이다.

---

## 6. 팀에서 기억할 한 문장
**A는 문서를 수정하면서 검증하고, B는 기존 프로젝트 분석을 바탕으로 코드와 구조를 수정하며 편입한다.**
