# docs/

This directory is a **reference archive**, not the primary source of truth for day-to-day implementation work.

For active implementation, prefer:
- root numbered `*.md` specs
- `client/web/`
- `server/`
- `shared/`
- root harness files such as `AGENTS.md`, `STATUS.md`, `WORKFLOW.md`, `AUTOMATION_SCOPE.md`, and `ASSET_STRATEGY.md`

## Current archived/reference groups

- `docs/reference/ai-workflow/`
  - AI workflow guidance, one-page summary, presentation material, and related reference notes
- `docs/reference/tamagotchi-llm/`
  - temporary borrowed planning/reference material

## Rule of thumb

If a document here becomes part of the active engineering loop, promote or rewrite it closer to the implementation surface instead of treating `docs/` as the default home.
