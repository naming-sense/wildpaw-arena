# AI 에이전트를 활용한 게임 개발 가이드라인
## 팀 공유용 1페이지 요약

## 1. 먼저 깔고 가야 할 생각
- AI를 블랙박스 자동 구현 도구처럼 쓰는 것은 권장하지 않는다.
- AI 활용의 핵심은 **내 지식베이스를 넓히는 것**에 있다.
- AI가 만든 내용은 가능한 한 내가 이해할 수 있어야 한다.
- 마이그레이션 개발 단계에서는 AI를 안 쓰고 직접 개발하는 편이 더 빠를 때도 있다.
- 그래도 장기적으로는 AI와 페어하게 개발하는 쪽이 생산성이 높아질 가능성이 크다.

---

## 2. 핵심 개념
AI 활용 방식은 두 가지로 나눈다.

- **A: 스펙 검증 개발**
- **B: 마이그레이션 개발**

가장 큰 차이는 **문제가 생겼을 때 무엇을 기준으로 수정하느냐**다.

- **A:** 스펙 문서를 수정하면서 검증
- **B:** 코드와 구조를 수정하면서 마이그레이션

한 줄 요약:
- **A는 스펙 검증용**
- **B는 마이그레이션용**

---

## 3. 기본 작업 흐름
A와 B 모두 아래 흐름을 기본으로 한다.

1. **요구사항 / 개발 계획을 md 파일로 정리**
2. **md 파일을 기준으로 작은 단위 구현**
3. **AI 리뷰, 인간 리뷰, 테스트 진행**
4. **수정사항 정리 후 다시 1번으로**

---

## 4. A와 B를 나누는 기준

### A(스펙 검증 개발)를 선택하는 경우
- 빠른 검증이 목적이다.
- 감각, 흐름, 기술 가능성을 먼저 보고 싶다.
- 코드가 나중에 버려져도 괜찮다.
- 문제가 생기면 요구사항, 개발 계획, 테스트 시나리오를 수정해서 다시 확인할 생각이다.

### B(마이그레이션 개발)를 선택하는 경우
- 메인 프로젝트에 넣어야 한다.
- 다른 개발자와 협업해야 한다.
- 코드 리뷰, 테스트, 유지보수가 필요하다.
- 문제가 생기면 실제 코드와 구조를 분석해서 고쳐야 한다.

---

## 5. 팀에서 기억할 한 문장
**AI는 딸깍 자동 구현 도구가 아니라, 내 지식베이스를 넓히면서 A는 스펙 검증하고 B는 마이그레이션하는 페어 개발 도구다.**
