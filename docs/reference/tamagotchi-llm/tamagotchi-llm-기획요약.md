# 다마고치 LLM 앱 기획 요약

## 1) 프로젝트 한 줄 정의
언리얼(클라이언트)과 WebSocket으로 실시간 통신하며, 사용자의 입력(텍스트/음성/터치)에 반응하는 **AI 다마고치 캐릭터**를 키우고 소통하는 앱.

## 2) 핵심 사용자 경험
- 캐릭터가 단순 챗봇이 아니라, 상태(배고픔/기분/친밀도/에너지)를 가진 존재처럼 반응
- 사용자가 터치하면 즉시 리액션(표정/애니메이션/대사)이 나타남
- 대답 대기 시간이 길지 않도록 **빠른 1차 응답 + 고도화 2차 응답** 제공
- 장기적으로는 사용자의 말투/의도를 더 잘 이해하고 개인화된 상호작용 제공

## 3) 주요 기능 아이디어
- **실시간 대화**: 텍스트/음성 입력 처리
- **리액션 엔진**: 터치/상황 이벤트 기반 행동 추천 및 실행
- **2단계 응답 파이프라인**:
  - 1차: 저지연 간단 응답(즉시성)
  - 2차: LLM 고도화 응답(맥락/감정/개인화 강화)
- **성장 시스템(다마고치성)**: 경험치, 친밀도, 성격 변화, 하루 루틴
- **개인화**: 사용자 선호/말투/자주 하는 행동 기반 적응

## 4) 지향하는 고도화 방향
- 사용자의 발화 의도 추론 정확도 향상
- 사용자 스타일을 반영한 말투/대화 전략
- 상황 인지형 행동 선택(무엇을 말하고 어떤 리액션을 할지 LLM이 결정)
- 음성 상호작용 품질 향상(말 따라하기/캐릭터 음성 일관성)

## 5) 실시간성 전략(핵심)
- 사용자 이벤트 수신 즉시:
  1. **즉시 ACK + 짧은 반응** (UX 지연 체감 감소)
  2. 백그라운드에서 **고품질 LLM 응답 생성**
  3. 준비되면 후속 메시지/행동으로 업데이트
- 즉, "빨리 반응하고, 뒤에서 더 똑똑하게 완성"하는 구조

## 6) MVP 범위 (권장)
1. WebSocket 기반 언리얼↔LLM 서버 양방향 통신
2. 텍스트 대화 + 터치 리액션
3. 1차/2차 응답 파이프라인
4. 기본 상태값(배고픔/기분/친밀도)과 간단 성장 로직
5. 로그/모니터링(응답속도, 오류율)

## 7) 성공 지표 예시
- 첫 반응(1차) 도착: 300ms~800ms 이내
- 최종 응답(2차) 체감 완료: 2~4초 이내 (일반 대화)
- 리액션 이벤트 성공률: 99%+
- 7일 재방문율/대화 지속시간 증가
