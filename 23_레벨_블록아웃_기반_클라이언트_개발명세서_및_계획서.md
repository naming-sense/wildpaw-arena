# 23. 레벨 블록아웃 기반 클라이언트 개발명세서 및 계획서

## 0) 문서 목적
본 문서는 `13_레벨_상세_블록아웃_명세서.md`를 바탕으로,
클라이언트 팀이 **바로 구현 가능한 레벨 시스템 개발 스펙**과 **실행 계획**을 제공한다.

핵심 목표:
1. 블록아웃 좌표/프리팹 명세를 런타임 데이터로 변환
2. Three.js + gameplay/ECS에서 레벨 로딩/충돌/오브젝트/모드 로직 구현
3. 맵별 QA/성능 기준과 튜닝 루프를 개발 프로세스로 고정

연동 문서:
- `13_레벨_상세_블록아웃_명세서.md`
- `12_레벨디자인_RnD_컨셉_구성요소_배치가이드.md`

## 1) 구현 범위 (MVP)

## 1-1. 대상 맵
- `NJD_CR_01` (3v3 크리스탈 러시)
- `HMY_SZ_01` (3v3 스위치 존)
- `FDD_PH_01` (5v5 페이로드 하울)

## 1-2. 클라이언트 책임
- 맵 데이터 로딩/검증
- 블록아웃 프리팹 인스턴싱(렌더 + 충돌)
- 모드 오브젝트(Core/Zone/Payload) 클라 연출/표시
- 미니맵/HUD 연동
- 레벨 디버그 도구(동선/시야/스폰 안전)

## 1-3. 완료 정의 (DoD)
- 세 맵 모두 로딩/진입/플레이 가능
- 지형 충돌, 엄폐, 목표 오브젝트 위치가 명세와 일치
- 모드별 목표 상태가 HUD/미니맵에 정확히 반영
- 평균 60FPS(권장), 저사양 30FPS 안정
- 레벨 관련 치명 버그(스폰 갇힘/충돌 누락/오브젝트 동기화 불일치) 0건

## 3) 데이터 설계 (블록아웃 -> 런타임)

## 3-1. 파일 포맷 원칙
- 좌표계/단위: `13번 문서`와 동일 (`1 unit = 1m`, origin map center)
- 렌더/충돌/게임플레이용 데이터를 하나의 맵 JSON으로 통합
- 사람이 읽기 쉬운 JSON 유지 + 빌드 단계에서 최적화 바이너리 변환 가능

## 3-2. 레벨 데이터 타입 (TypeScript)
```ts
export interface LevelMapDefinition {
  mapId: 'NJD_CR_01' | 'HMY_SZ_01' | 'FDD_PH_01';
  mode: 'CRYSTAL_RUSH' | 'SWITCH_ZONE' | 'PAYLOAD_HOWL';
  size: { width: number; height: number; };
  origin: { x: number; y: number; z: number; };
  spawnPoints: SpawnPoint[];
  objectives: ObjectiveDef[];
  prefabs: PrefabPlacement[];
  lanes: LaneDef[];
  tuning: LevelTuningDef;
}

export interface PrefabPlacement {
  id: string;                 // A_COV_01 등
  prefabCode: string;         // COV_H_3x1 등
  x: number; y: number; z: number;
  rotDeg?: number;
  extra?: Record<string, number | string | boolean>;
}
```

## 3-3. Prefab 카탈로그 (코드 고정)
`13번 문서 프리팹 코드`를 클라 카탈로그로 고정:
- `COV_L_2x1`, `COV_H_3x1`, `WALL_6`
- `BUSH_S`, `BUSH_M`, `BUSH_L`
- `RAMP_10`, `PAD_JUMP`
- `OBJ_CORE`, `OBJ_ZONE`, `OBJ_PAYLOAD_PATH`

카탈로그 필드 예시:
```ts
export interface PrefabCatalogItem {
  code: string;
  meshAsset: string;
  collider: 'box' | 'capsule' | 'ramp' | 'trigger';
  size: { x: number; y: number; z: number; };
  blocksLineOfSight: boolean;
  blocksProjectile: boolean;
  minimapLayer: 'cover' | 'bush' | 'wall' | 'objective';
}
```

## 3-4. 맵별 데이터 검증 규칙
공통:
- prefab id unique
- map bounds 밖 배치 금지
- 스폰 반경 8m 내 부쉬 금지
- 최소 통로폭 2.4m 미만 경고

NJD_CR_01:
- 코어 위치 `(0,0,0)` 고정
- A/B 스폰→코어 거리 시간차 0.5초 이내

HMY_SZ_01:
- 존 A/B 반경 4.5m
- 로테이션 타이밍 메타 포함(A45s, gap10s, B45s)

FDD_PH_01:
- Payload path 노드 P0~P6 순서 보장
- 체크포인트 CP1, CP2 좌표 유효

## 4) 런타임 로딩 파이프라인

## 4-1. 로딩 순서
1. `mapId` 결정 (매치 할당 payload)
2. 맵 JSON 로드 + schema validate
3. prefab catalog resolve
4. Three.js mesh group 생성
5. collision world 등록
6. objective layer 초기화
7. minimap bake 생성
8. match start 전 prewarm 완료

## 4-2. 실패 처리
- 데이터 검증 실패: `LEVEL_DATA_INVALID` 에러 + 로비 복귀
- 프리팹 에셋 누락: placeholder mesh 로드 + error telemetry
- 충돌 빌드 실패: 재시도 1회 후 매치 진입 차단

---

## 5) 레벨 시스템 상세 구현

## 5-1. 렌더 레이어 분리
- `LAYER_WORLD_STATIC`: 벽/엄폐/지형
- `LAYER_WORLD_INTERACTIVE`: 부쉬/점프패드/트리거
- `LAYER_OBJECTIVE`: 코어/존/카트 경로 시각화
- `LAYER_DEBUG`: 그리드, 콜라이더, LoS 레이

## 5-2. 충돌/지형 처리
- static box collider: wall/cover
- trigger collider: bush/objective/payload checkpoints
- ramp collider: 이동 보정(슬로프 정상 이동)

충돌 정책:
- 플레이어는 `COV_H/WALL` 통과 불가
- 투사체는 코드별 차단 여부 따름
- 부쉬는 통과 가능 + 시야 은닉 플래그만 제공

## 5-3. 라인 오브 사이트(LoS) 보조
- 클라에서 완전 판정은 안 하되, 시각 피드백용 raycast 제공
- 디버그 모드에서 `blocksLineOfSight`를 시각적으로 확인 가능

## 5-4. 스폰 안전 체크(런타임)
매치 시작 전 1회 체크:
- 스폰 중심 반경 8m 내 bush 없음
- 스폰 직선 LoS로 적 스폰 직시 가능 여부 검사
- 실패 시 로그 + fallback spawn offset 적용

## 5-5. 미니맵 투영
- world XY -> minimap UV 정규화
- 레이어별 심볼 고정:
  - wall/cover: 회색
  - bush: 녹색 반투명
  - objective: 모드색 강조
- 카메라 회전과 무관하게 북쪽 고정 모드(기본) + 회전 모드 옵션

## 6) 모드별 클라이언트 구현 명세

## 6-1. Crystal Rush (`NJD_CR_01`)
### 필요한 표시
- 코어 활성 상태
- 에너지 생성 타이머
- 점수 및 득점 알림

### 클라 로직
- 코어 주변 하이라이트 영역 렌더
- 에너지 픽업/드랍 연출
- 코어 근처 교차 사격각 디버그 뷰(테스트 모드)

## 6-2. Switch Zone (`HMY_SZ_01`)
### 필요한 표시
- 활성 존(A/B)
- 점령 진행률
- 존 전환 카운트다운

### 클라 로직
- 비활성 존은 desaturate 처리
- 전환 5초 전 경고 이펙트 점멸
- CLASH_MID 교전 유도 마커(옵션)

## 6-3. Payload Howl (`FDD_PH_01`)
### 필요한 표시
- 카트 진행률 바
- 체크포인트 도달 상태
- 공격/수비 phase 상태

### 클라 로직
- payload spline path 렌더(디버그 토글)
- 체크포인트 도달 시 구간 강조
- 병목 구간 진입/탈출 안내 VFX

## 8) 성능 예산

목표 (1080p, 권장 PC):
- static world draw calls: < 350
- objective + interactive draw calls: < 120
- triangle budget (map static): < 300k (블록아웃 단계)
- frame time budget: 16.6ms 중 level 렌더 < 5.5ms

최적화 원칙:
- static geometry merge
- instancing 가능한 프리팹 묶음 처리
- 부쉬/작은 오브젝트는 distance culling
