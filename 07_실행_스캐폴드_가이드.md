## 2) 현재 구현 범위
- Room 서버
  - WebSocket(Boost.Beast) **binary-only** 수신/송신
  - FlatBuffers Envelope decode (`HELLO/ACTION_COMMAND/INPUT/PING`) → 시뮬 입력 연결
  - `io_context` 멀티스레드 + `work_guard`
  - 전용 tick thread(고정 주기)
  - `pendingInputs` 워커큐 drain 후 authoritative tick 실행
  - FlatBuffers SnapshotPayload(`Base/Delta`) 전송
  - `CombatEventPayload`/`ProjectileEventPayload` 전송(사격/스킬/데미지/투사체 단계 이벤트)
  - `combat_rule_table` 기반 탄약/재장전/스킬 쿨다운/캐스트타임 적용
  - grid 기반 spatial interest filter 후 변경 엔티티 + 전투 이벤트 전송
  - `seq/ack/ackBits` 메타 트래킹(`RoomSession`) 연결
  - ack 기반 reliable retransmit queue + 유실 복구
  - 메시지 타입별 retransmit 정책 분리(Critical/Standard/BestEffort)
  - 세션별 `strand + write_queue` 송신 순서 보장
  - Prometheus `/metrics` 지표 노출
- Client netcode + gameplay loop
  - 입력 예측/복구 기본 함수
  - snapshot 보간 버퍼
  - FlatBuffers 기반 WebSocket 클라이언트 래퍼
  - `seq/ack/ackBits` 자동 송신/수신 추적
  - `WELCOME/SNAPSHOT/COMBAT_EVENT/PROJECTILE_EVENT/EVENT` binary payload decode
  - `RealtimeEcsRuntime`로 netcode/prediction/interpolation 결합
  - `ThreeCombatSceneAdapter`, `threeRealtimeLoop`로 Three.js 루프 직접 연결
- 프로토콜
  - `shared/protocol/fbs/wildpaw_protocol.fbs` 기준 단일 스키마
  - C++/TS 코드젠 산출물 포함

## 3) 실행 방법
```bash
cd server
cmake -S . -B build
cmake --build build -j
./build/room/wildpaw-room 7001 4 30 9100
```

클라이언트 측 스모크:
```bash
cd client/web
npm install
npx tsx ./scripts/bench-room.ts --url ws://127.0.0.1:7001 --clients 40 --duration-ms 4000 --input-interval-ms 50
npx tsx ./scripts/ecs-runtime-smoke.ts ws://127.0.0.1:7001 ecs-smoke
npx tsx ./scripts/interest-filter-smoke.ts ws://127.0.0.1:7001
```
